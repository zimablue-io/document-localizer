# Security Policy

## Reporting Vulnerabilities

We take security seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

1. **Do not** open a public GitHub issue for security vulnerabilities
2. Email us directly or use GitHub's private vulnerability reporting:
   - Go to the repository → Security → Advisories → "Report a vulnerability"

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### What We Promise

- Acknowledgment within 48 hours
- Regular updates on progress
- Credit in the security advisory (if you wish)

## Security Best Practices

### For Users

- Keep your AI backend software updated
- Only connect to AI servers you control
- Review the code if you have security concerns (it's open source!)

### For Developers

If you find a vulnerability and want to contribute a fix:

1. Open a draft pull request
2. Reference the security advisory
3. Follow our coding standards

## Scope

The Document Localizer desktop application processes files locally. The security of your AI backend (Ollama, LM Studio, etc.) is governed by their respective security policies.

## Out of Scope

- Vulnerabilities in third-party AI backends
- Social engineering attacks
- Physical security of your machine
- Security of your local network
